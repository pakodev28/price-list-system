# Система работы с прайс-листами и сметами

Веб-приложение для работы с прайс-листами поставщиков, каталогом товаров и сметами:
импорт данных из Excel с настройкой колонок, фоновый парсинг с прогрессом и
**автоматическое сопоставление позиций сметы с каталогом через ИИ** с цветовой
индикацией уверенности.

## Возможности

- **Поставщики**: CRUD, поиск по названию/ИНН, валюта.
- **Прайс-листы**: загрузка `.xlsx/.xls`, превью, настройка маппинга колонок, фоновый парсинг с прогрессом.
- **Каталог товаров**: артикул, наименование, ед. изм., группа — центральный узел сопоставления.
- **Проекты и сметы**: загрузка сметы из Excel, маппинг колонок (наименование, артикул, ед. изм., количество, цена материалов, цена монтажа), парсинг.
- **Сопоставление**: таблица позиций сметы, автосопоставление с каталогом, уверенность (зелёный/красный + %), ручная привязка и пометка «без соответствия».

## Стек

Django 5 + DRF · PostgreSQL 16 · Celery + Redis · python-calamine (Excel) ·
rapidfuzz + LLM (OpenAI-совместимый агент) · React + TypeScript + Vite · Docker Compose.

## Быстрый старт

```bash
cp .env.example .env
docker compose up --build
```

- Фронтенд: <http://localhost:5173>
- API + Swagger: <http://localhost:8000/api/docs/>
- Django admin: <http://localhost:8000/admin/> (примитивный «визуал» для данных)

Демо-данные и пользователь админки:

```bash
docker compose run --rm backend python manage.py seed_demo
docker compose run --rm backend python manage.py createsuperuser
```

После `seed_demo` откройте демо-смету в разделе «Проекты» и нажмите **«ИИ-сопоставление»**.

## ИИ-сопоставление

Пайплайн на каждую позицию сметы:

1. **Артикул** — точное совпадение нормализованного артикула → уверенность 1.0.
2. **Фаззи-шортлист** — `rapidfuzz` сужает каталог до ~5 кандидатов (локально, без сети).
3. **LLM-rerank** — кандидаты отправляются LLM (OpenAI-совместимый агент), который выбирает лучшего + уверенность.
4. **Фоллбэк** — если LLM выключен/недоступен, берётся фаззи-топ-1.

По умолчанию `LLM_ENABLED=False` — приложение полностью работает офлайн (артикул + фаззи).
Чтобы включить LLM, заполните в `.env`:

```env
LLM_ENABLED=True
LLM_BASE_URL=https://agent.timeweb.cloud/api/v1/cloud-ai/agents/<agent-id>/v1
LLM_API_KEY=<ключ>
LLM_MODEL=dashscope/qwen3.5-plus
```

Порог зелёный/красный задаётся `MATCH_THRESHOLD` (по умолчанию 0.75).

- Аутентификация (JWT) и разграничение доступа.
- Модуль сравнения цен поставщиков (offers по товару каталога) — продолжение замысла ТЗ.

- E2E-тесты (Playwright), CI, pre-commit, порог покрытия.
